１.　はじめに
	•	本書の位置づけ
本書は、納税者等クライアント（以下、「クライアント」という。）とe-Tax受付システムとの間でデータをやり取りするための、API仕様を定義するものである。

表 1.11用語の定義
用語
仕様書における定義
e-Tax
国税電子申告・納税システム
(参照URL https://www.e-tax.nta.go.jp/)
受付システム
納税者が税務署に申告書やその他の書類を提出する際に使用されるe-Taxのサブシステム。Web機能、認証機能、受付・審査機能、データ管理機能等を有する。
e-Taxホームページにてソフトウェア開発業者向けに仕様公開をしている。
送受信モジュール
HTTPS通信での送受信・リクエスト電文の生成・レスポンス電文に対するアクセスと、受付システムの画面IDから供給者ソフトウェアの画面IDへ変換する画面ID変換機能のインターフェースを提供するモジュール。
e-Taxホームページにてソフトウェア開発業者向けに仕様公開をしている。

利用者識別番号とパスワードでログインした利用者（法人利用者を除く）が、セキュリティ対策の観点より制限された一部機能を利用する際に追加で必要となる認証。マイナンバーカード又はその他の電子証明書を読み取ることによって認証が可能。

	•	全体像
本APIは、e-Taxで保持しマイページで画面表示しているデータモデルを「リソース」として公開し、リソースの操作機能(参照・追加・更新・削除)を提供するAPIである。
APIはHTTPプロトコルに基づいてインターフェースを提供し、リクエストの受信を受けて処理を開始し、結果をレスポンスで返却するものとなる。
クライアント、e-Tax、API間の全体像を図 1.21に示す。本APIの利用に当たって、利用者はe-Taxの利用者識別番号を保持し、e-Taxにログインしていることを前提とする。

	•	API概要
APIは以下の操作が可能である。
	•	利用者本人の情報の参照、更新、削除
	•	税理士による関与先納税者の情報の参照、更新

本書で仕様を提供するAPIの一覧は「資料1_API一覧」を参照。
「資料1_API一覧」に記載されている「リクエスト元の利用者の人格（個人・法人）」については2.3.1.2、「マスキング処理有無」については2.4.3を参照。

	•	API呼出しに必要な手続
API利用に当たっては、国税庁に利用を申請し、API認証情報（以下、「APIキー」という。）を発行される必要がある。利用申請については、e-Taxホームページを参照。

	•	APIの呼出し方
	•	リクエストヘッダ X-Etax-Api-Key に、国税庁から入手したAPIキーを設定しなければいけない。
	•	リクエストヘッダ Authorization パラメータに有効なトークンを設定しなければならない。トークンは、仕様公開されている送受信モジュール「GetValue関数」を使用することで取得可能である。送受信モジュールを使わない場合は、アプリケーションにて、受付システムログイン後に発行される引継ぎ情報をbase64でエンコードして設定する必要がある。なお、引継ぎ情報はAPIを呼び出す直前の操作まで受付システムで使用していた引継ぎ情報を設定することを推奨する。
	•	本APIはステートレスを採用しており、各リクエストは独立して処理される。サーバはクライアントの状態を保持せず、各リクエストにはAPIを処理するために必要な情報が全て含まれている。また、リクエストの順序性を問わないため、任意の順序で処理が可能。ただし、下記サンプルに記載しているように、別のAPIから事前に取得した情報をリクエストとして使用する場合もある。
	•	ブラウザからのAPI呼出しには対応していないため、サーバーサイド処理から呼び出すこと。必要に応じてBFF層を作成すること。

利用者本人のメッセージ情報を参照する場合を例として、呼出し方の一連の流れ（サンプル）を以下に記す。
	•	e-Tax受付システムにログイン
（e-Tax仕様書　受付システムインターフェース　SU00S010(認証)参照）
	•	GetValue関数（送受信モジュール）を用いて引継ぎ情報又はアプリケーションにて受付システムの引継ぎ情報（例：e-Tax仕様書　受付システムインターフェース　SU00S020(メインメニュー)のXBB020タグに記載されている引継ぎ情報）を取得。
	•	②で取得した引継ぎ情報をBase64変換しリクエストヘッダに設定。国税庁から発行されたAPIキーをリクエストヘッダに設定。別紙「APIインターフェース定義書（メッセージ一覧参照API）」を参考にその他のリクエストも設定し、メッセージ一覧参照APIを呼出し。
	•	返却されたレスポンス情報を基に、アプリケーションにてメッセージ一覧の画面を表示。利用者は、メッセージ一覧から参照したいメッセージを選択。
	•	利用者が選択したメッセージのメッセージID（メッセージ一覧参照APIのレスポンスから取得）を、メッセージ参照APIのリクエストに設定。その他のリクエストについても②③及び別紙「APIインターフェース定義書（メッセージ参照API）」を参考に設定し、メッセージ参照APIを呼出し。
	•	返却されたレスポンス情報を基に、アプリケーションにてメッセージ画面を表示。

	•	注意事項
	•	引継ぎ情報は受付システムとAPI間で共通のものが使用されるため、セッションも依存関係にあることに注意が必要。
例１：受付システムからログアウトした場合、ログアウト前に取得した引継ぎ情報を使用してAPIを呼び出すとエラーとなる。
例２：受付システムでAフォルダ内のメッセージ一覧を参照している状態で、フォルダパスワード認証APIを用いてBフォルダのパスワード認証を実施した場合、受付システムでAフォルダ内のメッセージを選択するとエラーとなる。

	•	プロトコル
通信プロトコルはHTTPSとする。通信可能なSSLのバージョンは、SSL Ver3.3（TLS1.2）とする。

	•	税理士による関与先納税者の情報参照
税理士が関与先の納税者の情報を参照する場合は、以下が必要。
	•	e-Taxにて税理士と納税者間で事前に委任関係を結ぶこと
	•	税理士が電子証明書による追加認証を実施すること
	•	関与先納税者の情報参照用のAPIを呼び出すこと（別紙「APIインターフェース定義書」参照）

税理士が関与先納税者の情報を参照する際の一連の流れの概要を以下に記す。
～事前作業（初回のみ必要）～
	•	納税者が税理士に対して委任関係の登録を行う。
（e-Tax仕様書　受付システムインターフェース　SU00S800（委任関係の登録）を参照又はe-Taxマイページから実施可能）
	•	税理士が委任関係納税者一覧（未承認一覧）画面から①の納税者との委任関係の承認を行う。
（e-Tax仕様書　受付システムインターフェース　SU00S820（委任関係納税者一覧）を参照又はe-Taxマイページから実施可能）
～追加認証（ログインする度に必要）～
	•	税理士が追加認証画面から電子証明書による追加認証を行う。
（e-Tax仕様書　受付システムインターフェース　SU00S760（電子証明書選択）及びSU00S770（閲覧申請）を参照又はe-Taxマイページから実施可能）
～税理士による納税者の情報参照～
	•	③を実施後の税理士の引継ぎ情報を取得し、APIのリクエストヘッダに設定。情報を参照したい納税者の利用者識別番号をAPIのパスパラメータに設定。その他のリクエストも別紙「APIインターフェース定義書」を参考に設定し、APIを呼び出す。
→パスパラメータに指定した納税者の情報が税理士側に返却される。

	•	制限事項
	•	利用可能時間
本APIの利用に当たっては、e-Taxが稼働中であることを前提とする。
e-Taxの利用可能スケジュールは、e-Taxホームページ（https://www.e-tax.nta.go.jp/info_center/index.htm）から確認する。
なお、利用可能時間外に API を実行した場合、ステータスコード 503 のエラーレスポンスを返却する。

	•	試験実施方法
利用可能なスケジュールについては国税庁に確認する必要がある。
APIの試験利用に当たっては、本番環境と同様に国税庁に利用を申請し、APIキーを発行される必要がある。
試験用APIシミュレータ等の公開はしていないため、2.2に記載されている検証環境を利用するか、個別でAPIシミュレータを使用する必要がある。
