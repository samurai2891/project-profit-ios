Ⅱ　インターフェイス仕様書
1.　CLICsvConvXMLインターフェイス

1.1　概要
CSV形式の財務諸表データ、勘定科目内訳明細書データ、および法人税申告書別表等データをXML形式(※1)に変換するためのインターフェイス。
(※1) e-Taxで受付可能なデータ形式(XML)のレベル3タグ配下

1.2　メンバー関数
(1)  CsvCheckToXbrl
HRESULT CsvCheckToXbrl (
[in] BSTR bstrCsvFilePath,  [in] BSTR bstrPropertyFolderPath, 
[in] BSTR bstrStyleVersion, [in] BSTR bstrNameConvListFilePath);

説明
財務諸表（HOT010（Ver2.0））に対応したCSVデータを検証する。なお、e-Tax標準科目への集約を行う勘定科目が存在する場合、勘定科目集約前後の情報をCSV形式で一覧に出力する。
引数
説明
bstrCsvFilePath
財務諸表CSVファイルのフルパス名（ファイル名含む）。なお、ファイル名の命名規則は以下とする。
「様式ID_財務諸表_様式バージョン.csv」
（例）HOT010_財務諸表_2.0.csv
bstrPropertyFolderPath
変換に使用するプロパティファイルを格納するフォルダパス名。
指定されたフォルダ配下には、以下のファイルが格納されていること。
・e-Tax標準科目和名プロパティ（CSV）
・e-Tax標準科目リスト（XML）
・変換情報プロパティ（XML）
bstrStyleVersion
検証する様式のバージョン(X.X)。
※対象帳票一覧に記載されている様式のバージョンを設定する。
bstrNameConvListFilePath
e-Tax標準科目へ集約を行った科目リストCSVファイルを出力するフルパス名（ファイル名含む）。
NULLまたは、空が指定された場合、出力は行わない。
なお、CSVデータには、カンマ区切りで"財務諸表CSVファイル内のレコード,集約前科目名,集約後科目名"を格納する。
（CSV出力例）
5,医薬品,薬品
8,車両,車両運搬具

説明

3.　詳細エラー情報の構成

3.1　概要
CLICsvConvXML.GetError、CLIXbrlResult. GetDetailErrorInfoで返却する詳細エラー情報の構成。

3.2　形式
AAAAA-BBBBBBB-CCCCCCCC

3.3　内容
各アルファベットは、以下のように設定し、繋ぎにはハイフンを使用する。

（１）CSV内容エラー
AAAA    ；CSVのレコード数 ※1
BBBBB　 ；CSVのカラム数
CCCCCCCC；エラーコード ※2
（例：0001-00002-HE9350E,0003:0004-00005-HE9440E）

（２）引数エラー
AAAA    ；9999
BBBBB　 ；誤りのある引数の順番
CCCCCCCC；エラーコード ※2
（例：CsvConvToXbrlのbstrCsvFilePathに誤りがあった場合
9999-00001- HCL9220E）

（３）その他エラー
AAAA    ；0000
BBBBB　 ；00000
CCCCCCCC；エラーコード ※2
（例：0000-00000- HCL9560E）

なお、2件以上エラーがある場合は、昇順にソートを行った後、カンマで区切って編集する。
（例：00007-0000005-HE40001E,00029-0000009-HE40009E）

処理中エラーが発生せず、すべて正常に終了した場合、空文字が取得される。

※1 財務諸表種別の組み合わせ不正等の複数レコードに跨ってエラーが発生した場合は、「開始レコード数：終了レコード数」を設定する。
※2 エラーコードの値は表 1のとおり。
