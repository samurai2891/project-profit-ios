## 2. テスト環境準備

### 2.1 必要なもの
- iPhone (iOS 17+) またはMac上のiOSシミュレータ
- TestFlightまたは開発用ビルド
- レシート画像 2〜3枚（レシートスキャン機能テスト用。コンビニ等の一般的なレシートで可）
- このテスト指示書（印刷またはデバイス上で参照）

### 2.2 アプリの初期状態
- 初回起動時、デフォルトカテゴリ（経費10件＋収益3件）が自動生成されます
- プロジェクト・取引はゼロの状態からスタートします
- **重要**: テストは Phase 1 から順番に実施してください。後のフェーズは前のフェーズで作成したデータに依存しています

## 3. アプリの画面構成

アプリは5つのタブで構成されています：

| タブ | アイコン | 機能概要 |
|------|---------|---------|
| ダッシュボード | ホーム | 全体の収支サマリー、プロジェクト別カード |
| プロジェクト | フォルダ | プロジェクト一覧・詳細・作成・編集 |
| 取引 | リスト | 取引一覧・新規作成・編集・削除 |
| レポート | チャート | 年度別レポート、カテゴリ内訳、プロジェクト別損益 |
| 設定 | 歯車 | カテゴリ管理、定期取引管理、データ管理、会計年度設定 |

---

## 4. 用語集

| 用語 | 説明 |
|------|------|
| プロジェクト | 収支を管理する案件単位（例：Webサイト制作、アプリ開発） |
| 取引 | 個々の収入または経費のエントリ |
| 収益 | 売上・サービス収入などの収入取引 |
| 経費 | 交通費・ツール代などの支出取引 |
| 配分（按分） | 1つの取引を複数プロジェクトに比率（合計100%）で分割すること |
| 日割り（プロラタ） | 月の途中でプロジェクトが開始/完了した場合の比例配分計算 |
| 定期取引 | 毎月/毎年自動的に繰り返し登録される取引 |
| 月次分割 | 年額の定期取引を12ヶ月に分割して毎月登録するモード |
| 終了予定日 | プロジェクトの完了予定日（推定プロラタ計算に使用） |
| 会計年度 | 集計の年度区切り（デフォルト4月始まり） |
| カテゴリ | 取引の分類（ホスティング、交通費、売上など） |
