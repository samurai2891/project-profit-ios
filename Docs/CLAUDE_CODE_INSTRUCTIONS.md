# 帳簿管理機能 組み込み指示書

## 鉄則

**コードを1行も書くな。まず調べろ。**

この指示書は3段階で構成される。
各段階の完了報告と私の承認なしに次の段階に進むことを禁止する。

```
段階A: 調査 → 報告 → 承認待ち
段階B: 設計 → 報告 → 承認待ち
段階C: 実装 → Phase毎にビルド確認 → 報告
```

---

## 背景

既存iOSアプリ「Project Profit」に帳簿管理機能を追加する。
帳簿のデータモデル・エクスポート処理は納品済み（Excel原本16ファイルと全セル検証済み）。
これを既存アプリに壊さず組み込むのが目的。

### 納品ファイル（Docs/ に格納済み）

| ファイル | 内容 | 変更可否 |
|---------|------|---------|
| `SPEC.md` | 全11台帳の列定義・計算ロジック・マスターデータ仕様 | 参照のみ |
| `master_schema.json` | 全台帳JSON定義（30KB） | **変更不可** |
| `LedgerModels.swift` | Swift Codableデータモデル（20KB） | **変更不可** |
| `LedgerExportService.swift` | CSV出力/読み込みサービス（16KB） | **変更不可** |
| `LedgerExcelExportService.swift` | Excel(.xlsx)出力サービス（29KB） | **変更不可** |
| `csv_templates/` | CSVテンプレート17ファイル | **変更不可** |

プロジェクトルートから以下を全て調べ、結果をそのまま出力せよ。


# 段階C: 実装

段階Bの承認後に実施。Phase毎にビルド確認し結果を報告すること。

## 実装ルール

### 納品ファイルは変更不可
`LedgerModels.swift`, `LedgerExportService.swift`, `LedgerExcelExportService.swift` は
一切変更しない。既存アプリとの差異は**ラッパー・変換レイヤー・Extensionで吸収**する。
問題がこれらのファイル側にある場合は修正案を提示し私の承認を得ること。

### 計算ロジック厳守

| 台帳 | 計算式 |
|------|--------|
| 現金出納帳 | 残高 = 前行残高 + 入金 - 出金 |
| 預金出納帳 | 残高 = 前行残高 + 入金 - 出金 |
| 売掛帳 | 残高 = 前行残高 + 売上金額 - 入金金額 |
| 買掛帳 | 残高 = 前行残高 + 仕入金額 - 支払金額 |
| 経費帳 | 合計 = 前行合計 + 金額 |
| 総勘定元帳(資産/経費/売上原価) | 残高 = 前行 + 借方 - 貸方 |
| 総勘定元帳(負債/資本/売上) | 残高 = 前行 - 借方 + 貸方 |

### Excel出力は数式で記入
残高列は計算値ではなくExcel数式（例: `=+E6+G5-F6`）

### インボイス版の列挿入位置
- 現金出納帳・預金出納帳: **勘定科目の後**
- 経費帳・総勘定元帳: **摘要の後**（相手科目の後ではない）
- 白色申告: **摘要の後**

### 仕訳帳の複合仕訳
続行行は月・日を空にする。`isCompoundContinuation = true` で判別。

### 勘定科目の台帳別サブセット
- 現金/預金/仕訳/総勘定元帳: 全51科目
- 売掛帳: 7科目 / 買掛帳: 7科目 / 経費帳: 4科目

### CSV出力
UTF-8 BOM付き（`\u{FEFF}` プレフィックス）
