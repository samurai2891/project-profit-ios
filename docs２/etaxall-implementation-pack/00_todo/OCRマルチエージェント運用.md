# OCR拡張 7人マルチエージェント運用

本ドキュメントは「レシート/請求書/領収書 OCR拡張」を安全に進めるための、開発プロセス上の役割分担を定義する。
アプリ実行時に複数エージェントを動かす機能は対象外。

## 役割

1. 要件固定エージェント
- スコープ/非スコープを固定し、受け入れ条件を定義する。

2. OCR抽出エージェント
- OCR抽出項目（書類種別、収支推定、カテゴリ、金額、日付）を実装する。

3. 推定ロジックエージェント
- Regex/Fallbackルール、曖昧時の既定値、信頼度算出を実装する。

4. UI統合エージェント
- OCR確認画面の種類切替、カテゴリ連動、保存連携を実装する。

5. 互換性保全エージェント
- 既存レシート経費登録フローの非回帰を確認する。

6. テストエージェント
- 新規テスト追加、既存テスト維持、回帰観点の網羅を確認する。

7. レビューゲートエージェント
- 最終差分レビュー、要件照合、リリース可否判定を行う。

## 完了条件

- 既存レシート機能を壊さず、請求書/領収書をOCRで取り込める。
- OCR確認画面で収支種別を自動初期化し、手動修正できる。
- 最終保存時に選択種別（収益/経費）が `PPTransaction` に反映される。
- 主要回帰テストが通過する。
